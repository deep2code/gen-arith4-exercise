# Gen Arith4 Exercise - 四则运算练习题生成器

一个用 Go 语言编写的智能四则运算练习题生成工具，支持生成包含加减乘除的混合运算题目,适用于小学数学练习。

## 功能特点

- 🎯 **智能出题**：自动生成合法的四则运算题目，确保：
  - 减法结果不为负数（被减数 > 减数）
  - 除法能够整除且结果为正数
  - 所有中间计算步骤和最终结果均为正整数
  - 最终结果不超过 200

- 📐 **递归生成**：采用递归算法自动生成表达式，每道题包含3-4个操作数，自动添加括号增加复杂度

- 🔢 **随机填空**：每道题目随机将一个操作数或结果替换为空白（`___`）

- 📄 **HTML 输出**：生成美观的 HTML 格式文件，支持：
  - 两列布局，适合打印
  - 响应式设计，适配屏幕显示
  - 打印优化，自动分页控制

## 技术实现

### 核心算法

1. **递归表达式生成**：采用递归算法自动构建表达式树
2. **运算符选择**：根据当前操作数值智能选择合法的运算符
   - 减法：确保被减数 > 减数
   - 除法：确保能整除且商为正数
3. **自动括号处理**：根据运算符优先级和结合律自动添加必要括号
4. **结果验证**：确保最终结果在 1-200 范围内

### 设计逻辑图

#### 整体生成流程

```
┌─────────────────────────────────────────────────────────────┐
│                    genRecursiveExercise()                    │
├─────────────────────────────────────────────────────────────┤
│  1. 设置固定深度 maxDepth = 2                           │
│                              ↓                              │
│  2. 调用 genExpr(0, maxDepth) 递归生成表达式                 │
│                              ↓                              │
│  3. 验证操作数数量 (3-4个，确保复杂度)                                  │
│                              ↓                              │
│  4. 添加等号和结果，构成完整练习题                            │
└─────────────────────────────────────────────────────────────┘
```

#### 递归表达式生成流程 (genExpr)

```
                        genExpr(depth, maxDepth)
                                    │
                    ┌───────────────┐
                    ↓               ↓               ↓
            depth ≥ maxDepth   
               返回操作数                                    ↓
                    │               │               │
                    └───────────────┘
                                    │
                        ┌───────────┴───────────┐
                        ↓                       ↓
              生成左子表达式              生成右子表达式
              genExpr(depth+1, maxDepth)  genExpr(depth+1, maxDepth)
                        │                       │
                        └───────────┬───────────┘
                                    ↓
                    选择有效运算符 op (确保结果合法)
                                    │
                                    ↓
                    根据优先级判断是否需要添加括号
                    组合: [左] op [右] → tokens
                                    │
                                    ↓
                            返回 ExprResult
```

#### 括号自动添加逻辑

```
┌─────────────────────────────────────────────────────────────┐
│                    括号添加判断规则                           │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  左子表达式需要括号:
│  └── OR (20%概率强制添加括号，增加题目多样性)                                          │
│  ├── 左子表达式不是单个操作数（包含运算符）                    │
│  └── 当前运算符优先级 > 左子表达式根运算符优先级                │
│                                                              │
│  右子表达式需要括号:
│  └── OR (20%概率强制添加括号，增加题目多样性)                                          │
│  ├── 右子表达式不是单个操作数                                 │
│  └── (当前优先级 > 右子优先级)                               │
│       OR (当前是减法/除法 AND 优先级相等)                      │
│                                                              │
│  核心函数:                                                   │
│  ├── getRootOperator() : 获取表达式的最外层运算符              │
│  ├── needsParenForOp(op, isRight) : 判断是否需要括号          │
│  └── wrapWithParen(tokens, needsParen) : 添加括号包装         │
│                                                              │
│  示例:                                                       │
│  ┌─────────────────────┬─────────────────────┐             │
│  │ 生成的表达式          │ 添加括号后           │             │
│  ├─────────────────────┼─────────────────────┤             │
│  │ 3 + 4 × 5          │ 3 + ❨4 × 5❩        │             │
│  │ 10 - 3 - 2         │ 10 - ❨3 - 2❩        │             │
│  │ 8 ÷ 2 ÷ 2          │ 8 ÷ ❨2 ÷ 2❩         │             │
│  │ 2 × 3 + 4 × 5      │ 2 × 3 + 4 × 5        │             │
│  └─────────────────────┴─────────────────────┘             │
└─────────────────────────────────────────────────────────────┘
```

#### 运算符优先级表

```
┌──────────┬────────────────────────────────────────────┐
│ 优先级   │ 运算符                                       │
├──────────┼────────────────────────────────────────────┤
│    2    │  × (乘法)    ÷ (除法)                      │
│    1    │  + (加法)    - (减法)                      │
└──────────┴────────────────────────────────────────────┘

运算规则:
- 优先级高的先计算
- 优先级相同时，从左到右计算（左结合）
- 减法和除法不满足结合律: (a-b)-c ≠ a-(b-c), (a÷b)÷c ≠ a÷(b÷c)
```

### 生成示例

```
输入: genExpr(0, 3)

递归过程示例:
depth=0: 生成 50 和 (depth+1=1)
  depth=1: 生成 30 和 (depth+1=2)
    depth=2: 返回操作数 12
  选择 op=×, 计算 30×12=360 → 超出范围，重新生成
    depth=2: 返回操作数 3
  选择 op=+, 计算 30+3=33
  组合: 30 + 3, 无需括号

depth=0: 选择 op=÷, 计算 50÷33 → 不整除，重新生成
  ...

最终生成: ❨30 + 12❩ × 2 = 84
```

### 表达式结构

程序通过递归自动生成多种表达式结构：

```
三元运算表达式:     a op1 b op2 c
带括号表达式:       (a op1 b) op2 c
                   a op1 (b op2 c)
复杂表达式:         (a op1 b) op2 (c op3 d)
```

### 符号说明

| 符号 | 含义 |
|------|------|
| ➕ | 加法 (+) |
| ➖ | 减法 (-) |
| X | 乘法 (×) |
| ➗ | 除法 (÷) |

## 开发指南

### 代码结构

- `TokenType`：定义标记类型（运算符、操作数、结果等）
- `Token`：表达式中的单个标记
- `Exercise`：练习题结构（标记数组）
- `ExprResult`：递归表达式结果结构
- `ArithmeticGenerator`：练习题生成器核心类

### 主要方法

- `genOperand()`：生成随机操作数
- `calc()`：执行二元运算
- `genValidOp()`：生成合法的运算符
- `getPrecedence()`：获取运算符优先级
- `genExpr()`：递归生成表达式（核心算法）
- `genRecursiveExercise()`：生成完整练习题
- `Generate()`：批量生成练习题
- `SaveToHTML()`：保存为 HTML 文件
- `wrapWithParen()`：根据条件给表达式添加括号
- `getRootOperator()`：获取表达式的根运算符
- `needsParenForOp()`：判断表达式作为子表达式时是否需要括号
